# 컨텍스트 문서 규칙

## 도메인 디렉토리 구조

```
context/{도메인}/
├── README.md          ← 도메인 개요 (배경, 문제, 사용자, 성공기준)
├── PROJECTS.md        ← 관련 GHE 레포 매핑
├── glossary.md        ← 용어 사전
├── architecture.md    ← 전체 구조 요약 + 주제 문서 링크 (인덱스)
├── status.md          ← 구현 추적 (AC별 ✅/⬜)
│
├── {주제}/README.md   ← 주제별 상세 정책/설계
└── ...
```

- 새 도메인 추가: `/new-context`
- `context/README.md`가 도메인 목록 인덱스 (GitHub 자동 렌더링)

## 문서 작성 규칙

- **architecture.md는 인덱스**: 전체 구조 요약 + 주제 문서 링크만 유지. 상세 정책은 주제 폴더로 분리
- **주제 폴더 추가 시**: 도메인의 `architecture.md` 주제 테이블과 도메인의 `README.md` 주제별 상세 테이블 모두 업데이트
- **주제 폴더명**: 주제명만 사용 (날짜 접두사 없음). 예: `템플릿-에디터/`, `팩토리-파이프라인/`
- **주제 문서 헤더**: 작성일, 수정일, 관련 레포를 포함
- **코드 반영 상태**: `status.md`에서 통합 추적 (인라인 태그 불필요)
- **변경 이력**: git에 위임. 문서에는 현재 상태만 유지
